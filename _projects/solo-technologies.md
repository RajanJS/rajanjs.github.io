## Context

At Solo Technologies, I work on a production mobile platform where customer workflows, release reliability, and long-term maintainability all matter at the same time. Much of the work is not appropriate to share in full detail, but the public story is clear: the product requires modern mobile architecture, strong production debugging, and thoughtful integration across payments, analytics, and real-time customer experiences.

## My Role

I serve as a Senior Mobile Engineer with responsibility across architecture, release readiness, platform modernization, and high-leverage feature delivery.

My work includes:

- React Native platform modernization
- native iOS integration in Swift for performance-sensitive modules
- Android and iOS build stability improvements
- AI-assisted customer workflows using real-time streaming
- release engineering and CI/CD improvements
- observability, production debugging, and performance tuning

## Technical Focus

This work is strongest proof of the kind of mobile engineering I want to keep doing:

- architecture that supports long-term iteration
- native integration when JavaScript-only solutions are not enough
- AI-assisted workflow design that reduces manual friction
- payment and subscription infrastructure
- crash reduction, runtime performance, and production reliability

## What This Proves

This project reflects my current level most directly. It shows that I can operate as more than a feature engineer: I can help shape platform direction, improve delivery systems, and ship mobile product work where reliability and maintainability are part of the job.
